# RosterFocus.app (native macOS)

A SwiftUI **menu-bar app** that does everything the Python CLI does, with a click-through
setup wizard — for people who'd rather not touch Terminal. It polls your calendar with
EventKit, decides which Focus should be on, and runs your Shortcut to toggle it; the Focus
then syncs to your iPhone via *Share Across Devices*.

It shares the **same config** as the CLI (`~/.config/roster-focus/config.json`), so the two
interoperate. (Don't run both the app and the launchd CLI agent at once — see the root README.)

## What it does for you

- **Calendar access** with a clean native prompt.
- **Setup wizard** — grant access → open Focus settings → import the Focus Shortcuts →
  edit rules (calendar/shortcut **dropdowns**, no typing exact names) → enable + launch at login.
- **Runs in the menu bar**: shows the current Focus and last action; enable/disable; Check Now.
- **Background scheduler** ticks every 60s using the exact same logic as the CLI (priority
  rules, keyword match, lead/trail, act-only-on-change, fail-loud toggling).

## What it still can't do (Apple limits)

- **Create a Focus** — no app can. The wizard opens System Settings → Focus so you make it once.
- **Set a Focus directly** — only Shortcuts can, so it runs your Shortcut.

## Build & run

Requires Xcode 14+ (developed on Xcode 26). The project is generated by
[XcodeGen](https://github.com/yonyz/XcodeGen) from `project.yml`; the generated
`RosterFocus.xcodeproj` is committed, so you can just open it.

**In Xcode:** open `app/RosterFocus.xcodeproj`, select the *RosterFocus* scheme, Run.

**From the command line:**
```bash
cd app
./scripts/build-local.sh        # builds + ad-hoc signs into app/build/…/RosterFocus.app
OPEN=1 ./scripts/build-local.sh # …and launches it
```
Ad-hoc builds run on this Mac; a copied/downloaded ad-hoc app needs **right-click → Open** once
(or `xattr -d com.apple.quarantine RosterFocus.app`).

Run the unit tests (the ported decision logic):
```bash
xcodebuild test -project RosterFocus.xcodeproj -scheme RosterFocus -destination 'platform=macOS' CODE_SIGNING_ALLOWED=NO
```

## Distribute to others (Developer ID + notarized)

With a paid Apple Developer membership:
```bash
TEAM_ID=YOURTEAMID ./scripts/package-notarize.sh
```
One-time setup (cert + `xcrun notarytool store-credentials ROSTERFOCUS_NOTARY`) is documented in
the script header. It archives → exports with Developer ID → notarizes → staples, producing an
app that opens with no Gatekeeper warning.

## Notes on the project

- **Not sandboxed, hardened runtime on.** The only entitlement is `app-sandbox = false` — the app
  must exec `/usr/bin/shortcuts` and read `~/.config`. Hardened runtime is applied at sign time
  (`--options runtime`) so the build is notarizable. Exec'ing the system `shortcuts` binary is an
  ordinary child process, so no `com.apple.security.cs.*` entitlement is needed.
- **Layout:** `RosterFocusCore` (framework) holds the pure logic + services and is unit-tested
  without a GUI host; `RosterFocus` (app) is the menu bar + wizard UI; `RosterFocusTests` covers
  the decider, config round-trip, and shortcut-list parsing.
